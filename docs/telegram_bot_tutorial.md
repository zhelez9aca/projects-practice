# Пошаговое руководство: как собрать Telegram-бота по документации TrackSwop

## 1. Идея вариативной части

Вместо абстрактного бота по учебной практике выбран бот-справочник по реальному проекту
`TrackSwop`. Это делает вариативную часть предметной:

- бот опирается на конкретный open-source проект;
- ответы можно проверять по README и исходникам;
- сайт и бот становятся дополнением к основному проекту по дисциплине.

## 2. Какие источники использованы

Перед реализацией были изучены:

1. `README.md` репозитория TrackSwop;
2. `src/model/services/providers/spotify/README.md`;
3. `src/model/services/registry/service_registry.py`;
4. `src/model/store/token_store.py`;
5. код провайдеров `spotify`, `vk`, `local`.

Эти файлы дали фактическую основу для ответов бота.

## 3. Как устроен контент

В `src/practice_bot/content.py` контент вынесен в отдельный слой:

```python
KEY_FEATURES = [
    "динамическая генерация форм на основе спецификаций провайдера;",
    "шифрование токенов через Fernet и хранение в tokens.json;",
    "реестр провайдеров для Spotify, VK Music и локальных файлов;",
]
```

Там же хранятся:

- описание проекта;
- целевая аудитория;
- цели;
- особенности провайдеров;
- шаги установки;
- список источников.

## 4. Маршрутизация запросов

В `src/practice_bot/engine.py` пользовательский текст сопоставляется со сценарием:

```python
routes = (
    (("о проекте", "/about", "trackswop"), self._about),
    (("возможности", "функции"), self._features),
    (("архитектура", "mvvm"), self._architecture),
    (("spotify",), self._spotify),
)
```

Так бот отвечает не в свободной форме, а строго по заранее проверенным разделам.

## 5. Telegram-слой

`src/practice_bot/telegram_app.py` подключает движок к `aiogram`:

```python
@dispatcher.message()
async def on_message(message: Message) -> None:
    reply = engine.respond(message.text or "")
    await message.answer(reply.text, reply_markup=_build_keyboard(reply.suggestions))
```

Telegram-слой остаётся тонким адаптером: вся предметная логика находится вне него.

## 6. Web-режим

Для локальной демонстрации без Telegram используется `src/webapp.py`. Он:

- раздаёт сайт из `site/`;
- публикует API `/api/chat`;
- позволяет использовать тот же набор ответов в браузере.

## 7. Что именно умеет бот

Бот отвечает на команды:

- `О проекте`
- `Пользователи`
- `Цели`
- `Возможности`
- `Архитектура`
- `Провайдеры`
- `Spotify`
- `VK Music`
- `Local Files`
- `Установка`
- `Безопасность`
- `Тесты`
- `Статус`
- `Источники`

## 8. Почему такой подход корректен

Это решение соответствует задаче лучше, чем прежняя версия по общим материалам практики,
потому что:

- сайт теперь действительно посвящён основному проекту по дисциплине;
- бот отвечает по документации и исходникам этого проекта;
- содержимое можно проверить по первичным источникам.

## 9. Возможные улучшения

- вынести контент в JSON и использовать его и в Python, и в JS без дублирования;
- добавить ссылки на конкретные файлы и строки источников в ответы бота;
- подключить режим вопрос-ответ поверх векторного поиска по документации TrackSwop.
